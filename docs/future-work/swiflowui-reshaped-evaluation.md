# SwiflowUI — Reshaped Evaluation

> **Date:** 2026-06-25 · **Subject:** [Reshaped](https://github.com/reshaped-ui/reshaped)
> (site: [reshaped.so](https://www.reshaped.so/)) · **Question:** Should SwiflowUI learn from
> Reshaped, and if so, which specific patterns?
>
> Companion to [`swiflowui-1.0-roadmap.md`](swiflowui-1.0-roadmap.md). The roadmap records the
> *decisions* (the M8 milestone + rejected items); this doc preserves the *reasoning* — the
> grid, the scores, the sources, and the mapping — so the conclusions can be audited or
> revisited without re-running the evaluation.

## TL;DR

- **Verdict: clearly worth learning from.** Weighted score **4.62 / 5** (gate was 3.5).
- The surprise: Reshaped is **build-time CSS variables**, not JS-runtime theming. The thing we
  expected to disqualify it (CSS-in-JS, JS breakpoint branching that would violate our
  media-first stance) it *doesn't do*. Its token layer is architecturally convergent with
  SwiflowUI.
- **License gate passes wide open:** MIT, code legally reusable (it was a paid/closed product
  for ~5 years before the OSS relaunch).
- **The output is a steal-list, not adopt/reject.** The high-value steals cluster on the
  **token-generation / correctness layer** — exactly the seam SwiflowUI 1.0 left hand-authored.
  They became the proposed **M8 (1.1) — Token correctness & generation** milestone.

## What Reshaped is

A mature React + TypeScript design system. Ships as ESM, tree-shaken; styling is **CSS Modules
+ a `classNames` utility** (build-time, near-zero runtime), with all design tokens as CSS
custom properties prefixed `--rs`. Theming is a React provider at runtime (`data-rs-theme` /
`data-rs-color-mode` attributes for SSR), but the *substrate* is pure CSS cascade. v3.10.3,
~21k weekly npm downloads, ~2k GitHub stars. MIT.

## The evaluation grid

Two things are scored at once: *is it good?* **and** *is it transferable* given the
React→Swift/Wasm gap. Weights lean toward what SwiflowUI can actually absorb, not toward what
makes Reshaped impressive in React-land.

| # | Dimension | Weight | Transferability lens |
|---|-----------|:------:|----------------------|
| 1 | Token & theme architecture | 20% | Concept ports; React provider mechanism doesn't — we do it in CSS media features |
| 2 | Styling mechanism & build model | 15% | Maps onto our two seams: stateless `.sw-*` sheet vs scoped `@Component` |
| 3 | Component API & composition | 15% | Naming + composition ideas port; React idiom doesn't |
| 4 | Accessibility & contrast discipline | 15% | Directly relevant to the soft-tint contrast failure |
| 5 | Responsive / adaptive model | 10% | Must stay media-first; steal CSS-resolution, not viewport-only worldview |
| 6 | Layout primitives & spacing | 8% | Port candidate, but weigh against shipped SwiftUI-idiom API |
| 7 | Source readability & learning value | 7% | Meta-dimension |
| 8 | Maturity & maintenance | 5% | Context only |
| 9 | License & reuse rights (**gate**) | 5% | Decides "port code" vs "study ideas" |

**Scoring:** each dimension 0–5; quality score = Σ(score × weight).

**Two decision rules:**
1. **Quality gate** — worth studying at all? Weighted score ≥ **3.5 / 5**.
2. **Steal threshold (the real output)** — per pattern, lift it only if transferability is
   🟢/🟡 **and** it fills a *known SwiflowUI gap*. Deliverable = a ranked steal-list, each
   item tagged port-as-is / adapt-concept / inspiration-only.

**Transferability legend:** 🟢 Direct · 🟡 Adapt · 🔵 Inspiration · ⚪ N/A.

from**, and unusually so, because the architecture doesn't fight SwiflowUI's constraints.

## The steal-list (ranked by transferability × known gaps)

**🟢 High — port directly, fills a known gap**

1. **On-background contrast tokens** (autogenerated, WCAG/APCA). *The* one. Directly fixes the
   light-mode soft-tint contrast failure: compute the text token *per background* against a
   contrast criterion instead of hand-curating `--sw-<semantic>-strong`. Highest-value steal.
2. **Theme generator** (`generateThemeColors`-style): one primary hex → full light+dark OKLCH
   palette. Adds tooling SwiflowUI lacks (tokens are hand-authored).
3. **Single flexible `View` primitive** — a design *lesson* to weigh, not necessarily adopt
   (see rejection below).

**🟡 Medium — adapt the concept**

4. **Scoped `Theme` fragment** (brand-a-subtree token override) → maps onto a subtree `--sw-*`
   re-point via existing `CSSEntry.group(prefix:)` machinery.
5. **Headless/utilities split** — separate interaction logic from styling at the package level.
   Architectural lesson, not a code port.
6. **OKLCH as default color space** — perceptually uniform; makes contrast-correct palette
   derivation tractable. Pairs with #1/#2.

**🔵 Inspiration only**

7. Smart polymorphic Button (`<a>` vs `<button>` by `href`).
8. Per-component a11y regression tests as a discipline.
9. `--rs` prefix discipline (we already do `--sw-` — validation, not new).

## Mapping to the 1.0 roadmap

The roadmap (M1–M7) is **complete** — it shipped the media-feature **response** system but
left base token values hand-authored. The steal-list therefore lands almost entirely as
**new 1.1 theming-tooling**, with two deliberate rejections.

| # | Steal item | Disposition |
|---|-----------|-------------|
| 1 | On-background contrast tokens | 🟢 **Adopt → M8 (1.1).** Structurally fixes soft-tint failure. |
| 2 | Theme generator | 🟢 **Adopt → M8.** |
| 3 | OKLCH color space | 🟢 **Adopt → M8** (pairs with #1/#2). |
| 4 | Scoped `Theme` region | 🟡 **Adopt → M8** (medium). Extends "components read tokens"; doesn't fight it. |
| 5 | Single `View` primitive | 🔴 **Reject.** Breaking change vs shipped SwiftUI-idiom `VStack`/`HStack`/`Grid`/`ZStack` (M1). Keep only the lesson: *lean on modifiers, not a primitive per layout.* |
| 6 | Headless/utilities split | ⚪ **Defer (premature).** Revisit if `Sources/SwiflowUI` outgrows its footprint. |
| 7 | Polymorphic Button | ⚪ **Lean reject.** `Button` vs Router `Link` separation is cleaner than overloading. |
| 8 | Per-component a11y tests | ✅ **Already covered** (Verification: a11y smoke + Playwright `emulateMedia`). |
| 9 | `--rs` prefix | ✅ **Already covered** by `--sw-`. |

## Caveats & UNVERIFIED items

- **Don't let Reshaped's breakpoint model narrow ours.** Their responsive is *viewport* media
  queries (s/m/l/xl). SwiflowUI's media-first stance is broader — `light-dark()`,
  `prefers-contrast`, `prefers-reduced-motion`, `prefers-reduced-transparency`, `color-gamut`.
  Steal the CSS-resolution discipline, not the viewport-only worldview.
- **UNVERIFIED** (researcher's web sandbox couldn't read the repo tree directly — WebFetch/Bash
  denied): exact spacing-scale step values; repo commit recency / contributor count; the
  vanilla-extract attribution seen in an early generic search result was **not** Reshaped-
  specific (Reshaped's own docs describe CSS Modules + `classNames`).
